Engineering with AI & LLMs

I don't just use AI assistants — I treat LLMs as a programmable layer of the stack and study how to drive them reliably. Actively leveling up on:

  • Prompt engineering for code — task decomposition, few-shot & schema-constrained structured outputs, eval-driven prompting, and reducing hallucination on programming tasks.
  • Agentic workflows — multi-step agents with tool use, planning/execution loops, and verification gates; orchestrating parallel sub-agents for research, review, and refactoring.
  • AI-assisted development in practice — daily work with Claude Code and agentic harnesses to plan, generate, review, and test code under TDD and code-review loops.
  • LLM application patterns — the Claude API (Messages, tool use, streaming, prompt caching), RAG, cost-aware model routing, and MCP-style tool servers.

🎯 Goal: ship production software where AI agents handle a large share of implementation under tight, human-defined specs, tests, and review gates — faster, without sacrificing quality.
